Rendering Approach: How should we achieve the realistic look?

The current globe uses Canvas 2D (pixel-by-pixel texture mapping at 38% opacity over a gradient). To get the quality in your reference, we need to decide the rendering strategy.

A

Migrate to WebGL / Three.js

Best visual quality. Real GPU-accelerated 3D sphere with proper texture mapping, atmospheric shaders, city lights layer, Fresnel edge glow, and smooth rotation. This is how the reference images are built.

Pros

  • Photo-realistic earth with multiple texture layers (day, night, clouds, bump)
  • Real atmospheric glow via shaders
  • GPU-accelerated — smooth even at high resolution
  • Aligns with PRD goal (WebGL with Three.js)

Cons

  • Significant rewrite of the renderer (~500 lines)
  • Adds Three.js dependency (~150KB gzipped)
  • All existing Canvas 2D drawing code needs porting
B

Enhanced Canvas 2D

Incremental improvement. Boost the existing renderer: full-opacity texture, better lighting simulation, painted atmospheric glow, composite night-lights layer. Stays in Canvas 2D.

Pros

  • No new dependencies
  • Smaller diff — evolve existing code
  • Simpler mental model

Cons

  • Still pixel-by-pixel — slow at high resolution
  • Can't achieve true atmospheric shader effects
  • Night/day blending will be approximate at best
  • PRD actually calls for WebGL — this defers that
C

Hybrid: WebGL sphere + Canvas 2D overlays

Best of both. Use WebGL (Three.js) only for the sphere rendering (texture, atmosphere, lighting). Keep Canvas 2D for markers, paths, arcs, and callouts overlaid on top.

Pros

  • Photo-realistic sphere with shaders
  • Preserves all existing overlay code (markers, paths, regions)
  • Smaller migration surface — only sphere body changes
  • Callouts naturally work as HTML/Canvas overlays

Cons

  • Two rendering contexts to coordinate
  • Still adds Three.js dependency
  • Slight alignment complexity between WebGL and Canvas layers

My recommendation: Option C (Hybrid) — gets the visual quality where it matters most (the earth itself) while preserving the working overlay system. The callout system you want is actually easier as HTML/Canvas overlays than as 3D objects.
